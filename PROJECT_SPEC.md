# AgentPilot 项目规格书

## 1. 项目名称

AgentPilot

## 2. 项目一句话定义

一个开源的 AI Agent 创建、编排、知识库问答、工具调用和评测平台。

## 3. 项目目标

让用户通过图形化界面创建自己的智能体，并为它配置知识库、工具和执行流程。系统负责运行 Agent、记录每一步过程、支持长任务、展示引用来源，并对不同模型和策略进行评测。

## 4. 核心需求

1. LLM
   - 支持多模型接入。
   - 支持流式输出、重试、超时、fallback。
   - 支持 token 和成本统计。

2. RAG
   - 支持文档导入、解析、切块、索引和检索。
   - 支持多模态提取：文本、图片、表格。
   - 支持多模态 Embedding：文本向量和图片向量统一空间。
   - 支持跨模态检索：用文本查询图片、用图片查询文本。
   - 支持引用来源回传。
   - 支持检索结果重排。

3. 切面
   - 支持日志、鉴权、限流、审计、追踪等横切能力。
   - 支持在 Agent 执行过程中统一注入这些能力。
   - 支持对每一步执行做埋点和可观测统计。

4. 向量
   - 支持多模态 embedding 生成与存储（文本 + 图片）。
   - 支持向量相似度检索和跨模态检索。
   - 默认基于 PostgreSQL + pgvector 落地。
   - 多模态 Embedding 模型支持 CLIP、Cohere Multimodal 等统一向量空间模型。

## 5. 适用场景

- 企业客服问答
- 资料研究与总结
- 代码审查和 PR 分析
- 周报、日报、报告生成
- 内部知识库助手

## 6. 核心价值

1. 用户可以自己创建智能体。
2. 智能体可以接入知识库和外部工具。
3. 每次执行都有完整日志和引用。
4. 可以比较不同模型、提示词和检索策略。
5. 适合作为 GitHub 开源项目和面试项目。

## 8. 数据库推荐

主库使用 PostgreSQL + pgvector。

原因：
- 业务数据和向量数据可以统一管理。
- 适合开源项目快速落地。
- 后期可以平滑扩展到 Qdrant 或其他独立向量库。

## 10. MVP 范围

第一版只做最核心的功能：

1. Vue 前端基础页面
2. FastAPI 后端基础服务
3. 登录和用户体系
4. Agent 创建和列表
5. 知识库上传和检索
6. 简单 Chat Agent
7. 流式输出
8. 执行日志展示
9. Docker Compose 一键启动

## 11. 非目标

第一版不做：

- 多租户复杂计费
- 复杂审批流
- 大规模分布式调度
- 复杂 BI 报表
- 过多第三方平台集成

## 13. 开发原则

- 先做可运行，再做完整。
- 先做主流程，再补增强功能。
- 先用 PostgreSQL + pgvector 起步。
- 先做单体，后考虑拆分服务。
- 所有 Agent 执行必须可追踪。

## 15. 当前推进顺序

1. 先搭项目骨架。
2. 再做登录和基础页面。
3. 再做 Agent 创建和运行。
4. 再做知识库和 RAG。
5. 再做工具系统。
6. 再做评测和可观测性。

## 16. 第一版验收流程

第一版完成后，需要能跑通下面这条链路：

1. 创建知识库。
2. 上传文档。
3. 系统切分文档并生成向量。
4. 输入问题做检索测试。
5. 前端展示检索结果：
   - 文档来源
   - 命中文档片段
   - 相似度分数
6. 创建 Agent。
7. Agent 绑定知识库。
8. 执行任务。
9. Agent 返回回答、引用来源和执行轨迹。

## 17. 当前实现状态

截至当前版本，项目已经从最初 MVP 推进到可演示的前后台智能体工作台：

1. 前后台分离
   - 后台用于模型配置、知识库、工具和智能体管理。
   - 前台 `/portal` 是独立用户体验页，不显示后台菜单。
   - 后台可点击“进入前台”，前台可点击“进入后台”。

2. 模型配置
   - 支持多个 OpenAI-compatible 模型配置。
   - 支持默认模型和连通性测试。
   - 后端大模型生成超时已调长，适合更复杂回答。

3. 知识库
   - 支持知识库创建、文档上传、切块、检索测试。
   - 支持多模态提取：PDF 中的图片和表格也能被解析和向量化。
   - 支持跨模态检索：用文本查询图片内容，用图片查询相关文本。
   - Agent 运行时会检索绑定知识库，并返回引用来源（含图片引用）。

4. 应用工具
   - 支持工具列表接口。
   - 创建智能体时可以绑定工具。
   - 运行时根据用户问题调用已绑定工具。
   - 当前内置当前时间和订单查询示例工具，后续可替换为真实业务 API。
   - HTTP 工具管理页面，支持创建、编辑、删除和测试。
   - 智能体可绑定数据库持久化工具，运行时优先调用数据库工具，内置工具作为 fallback。

5. 智能体管理
   - 支持编辑智能体名称、描述、系统提示词、模型、知识库和工具。
   - 支持复制和软删除智能体。
   - 支持发布 / 下线智能体，前台只展示已发布智能体。

6. 运行历史
   - Run 列表页和单次运行详情页。
   - 展示输入、回答、引用来源、执行步骤、模型、状态和耗时。
   - 展示工具调用明细（工具名称、成功/失败状态、返回内容）。

7. 回答展示
   - 支持 `<think>` 思考过程折叠。
   - 支持 Markdown 代码块灰色显示。
   - 支持清理标题符号和分隔线。
   - Markdown 渲染结果经 DOMPurify 消毒，防止 XSS 攻击。

8. 流式输出
   - 前台 `/portal` 使用 SSE（Server-Sent Events）流式显示回答。
   - 后台 `/chat` 仍使用非流式接口。

9. 使用体验
   - 支持深色 / 亮色主题切换。
   - 新增 `/guide` 使用教程页面。
   - Docker Compose 可一键启动前端、后端、PostgreSQL、Redis 和 MinIO。
   - 前端路由 404 兜底，未匹配路径重定向到首页。

## 18. 后续自我发展规划

第一版之后，项目进入“自我发展”阶段，但这不是让系统直接自主失控扩展，而是让系统基于历史运行数据持续优化自身。

规划方向：

1. 运行复盘
   - 自动分析每次 Agent 执行记录。
   - 找出失败步骤、低命中检索、低质量回答和高成本调用。

2. 参数优化建议
   - 对 prompt、chunk 大小、top_k、reranker、模型选择给出建议。
   - 根据历史效果推荐更合适的配置。

3. 知识库优化
   - 自动识别低质量文档和无效切块。
   - 建议重建索引或调整切块策略。

4. 任务模板演化
   - 从成功的任务中沉淀模板。
   - 形成可复用的 Agent 配置版本。

5. 评测闭环
   - 每次改动后自动跑评测集。
   - 对比优化前后的回答质量、引用命中率和成本。

6. 受控自我发展
   - 只允许系统提出建议，不直接修改核心配置。
   - 所有自动调整需经过用户确认或管理员审批。

## 19. 第二版规划

第二版的目标不是继续堆演示页，而是把第一版工作台推进成可长期使用的管理平台。

### 19.1 目标

- 智能体可编辑、可复制、可删除、可发布。
- 工具系统从示例升级为真实可配置能力。
- 后台具备基础登录和权限。
- 运行记录可追踪、可回看、可定位失败原因。
- 前台支持更自然的流式对话体验。

### 19.2 已完成

1. 智能体编辑与发布
   - 编辑名称、描述、提示词、模型、知识库、工具。
   - 发布 / 下线状态。
   - 前台只展示已发布智能体。

2. 工具系统产品化（HTTP 工具部分）
   - 工具管理页面。
   - HTTP 工具配置（URL、Method、Headers、Query、Body、触发关键词、超时）。
   - 工具测试接口。
   - 智能体绑定数据库持久化工具。

3. 运行历史
   - Run 列表页。
   - Run 详情页。
   - 输入、回答、引用、工具调用明细、耗时、模型信息。

4. 前台体验增强
   - SSE 流式输出。
   - 更清晰的回答排版（思考过程折叠、代码块、Markdown 渲染 + DOMPurify XSS 防护）。

### 19.3 待推进

1. 工具系统增强
   - SQL 工具配置。
   - 工具调用日志表。
   - 鉴权配置页。

2. 基础权限
   - 管理员登录。
   - 后台鉴权。
   - 前台匿名体验。

3. 运行历史增强
   - 失败原因和重试入口。

4. 前台体验增强
   - 回答重试。
   - 更清晰的错误提示。
   - Mermaid.js 图表渲染：智能体回答中的 mermaid 代码块自动渲染为流程图、时序图、甘特图等可视化图表。

5. 检索增强
   - 文档级管理和单文档删除。
   - OCR 支持扫描件 PDF。
   - ~~多模态 Embedding：支持 CLIP 等统一向量空间模型，文本和图片在同一向量空间中检索。~~（v0.2.0 已完成）
   - ~~图片提取：PDF 中的图片独立提取、切块和向量化。~~（v0.2.0 已完成）

6. 对话与智能体增强
   - 对话记忆管理：查看历史会话列表、删除会话、会话搜索。
   - 上下文压缩（滑动窗口 + 摘要）：长对话自动压缩早期上下文，Token 使用率达到上下文窗口 60% 时触发，保留最近 6 轮原文，早期对话由 LLM 生成摘要替换，摘要缓存到数据库避免重复生成。详见 `docs/roadmap-v2.md` P6 详细设计。
   - 多知识库绑定：一个智能体绑定多个知识库，检索时合并结果。
   - 智能体导入/导出：JSON 格式导出智能体配置，跨环境迁移。
   - 回答反馈：用户对回答点赞/点踩，记录到数据库。
   - Prompt 模板库：预置常用系统提示词模板，一键套用。

### 19.4 第二版验收预期

第二版完成后，应能够跑通：

1. 管理员登录后台。
2. 创建并发布智能体。
3. 绑定知识库和真实工具。
4. 前台选择已发布智能体。
5. 用户提问。
6. 系统按需调用知识库和工具。
7. 前台实时显示回答。
8. 后台查看运行历史和详情。
