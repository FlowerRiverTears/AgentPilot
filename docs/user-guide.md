# 用户使用教程

## 访问地址

- 后台工作台：http://localhost:5173
- 前台体验：http://localhost:5173/portal
- 使用教程：http://localhost:5173/guide

## 第一步：配置模型

进入后台的“模型配置”页面。

填写：

- 配置名称
- Base URL
- API Key
- 默认模型名

填写完成后点击“测试”。测试通过后再创建智能体。

## 第二步：创建知识库

进入“知识库”页面。

操作：

1. 创建知识库。
2. 选择知识库。
3. 上传 PDF、Markdown 或文本文件。
4. 输入问题做检索测试。

知识库适合放稳定资料，例如产品手册、FAQ、制度文档。PDF 会提取其中可读文本后再切块；扫描件或图片型 PDF 可能无法提取文字。

## 第三步：配置工具（可选）

如果智能体需要查询实时业务数据（如订单、库存、天气），可以进入"工具管理"页面创建 HTTP 工具。

操作：

1. 点击"新增工具"。
2. 填写工具名称和描述。
3. 配置 URL、Method（GET / POST）、触发关键词。
4. 按需配置 Headers、Query 参数、Body 模板和超时时间。
5. 点击"测试"验证工具是否可用。

创建工具后，在创建智能体时绑定该工具即可。运行时如果用户问题匹配触发关键词，系统会自动调用工具并将结果交给大模型。

## 第四步：创建智能体

进入“智能体后台”页面。

填写：

- 名称
- 描述
- 模型配置
- 系统提示词
- 知识库
- 应用工具

创建后，智能体会出现在前台。

## 第五步：前台体验

点击后台右上角“进入前台”，或访问：

```text
http://localhost:5173/portal
```

前台只展示用户对话体验，不显示后台管理菜单。

前台使用流式输出，回答会逐字显示，体验更自然。

刷新前台页面后，最近会话会保留在浏览器本地，并继续作为多轮上下文参与回答。

## 第六步：查看运行过程

后台“运行测试”页面可以选择智能体并执行任务。

运行后会看到：

- 回答
- 执行过程
- 引用来源

也可以进入“运行历史”页面，查看最近的运行记录和单次运行详情。

## 第七步：管理对话记忆

前台支持对话记忆管理：

1. 在前台左侧可以看到历史会话列表。
2. 点击会话可以继续对话。
3. 可以删除不再需要的会话。
4. 可以通过搜索框搜索历史对话内容。

刷新页面后，最近的会话会自动恢复。

## 第八步：使用反馈功能

在前台对话中，每条回答下方有点赞和点踩按钮：

1. 点击 👍 表示回答有帮助。
2. 点击 👎 表示回答不满意。
3. 反馈会记录到数据库，管理员可在后台查看反馈统计。

## 第九步：导入导出智能体

在智能体后台页面：

1. 导出：点击智能体操作菜单中的"导出"，下载 JSON 配置文件。
2. 导入：点击"导入智能体"，上传 JSON 配置文件，即可在其他环境中复用智能体配置。

## 第十步：使用 Prompt 模板

创建智能体时：

1. 在系统提示词输入框下方可以看到预置的 Prompt 模板。
2. 点击模板即可一键套用到系统提示词。
3. 模板包含常见场景的提示词，如客服助手、文档问答、代码审查等。

## Mermaid 图表

智能体回答中如果包含 Mermaid 代码块（如流程图、时序图、甘特图），会自动渲染为可视化图表。

## 回答格式

系统会自动处理：

- `<think>` 思考过程折叠显示。
- Markdown 代码块显示为灰色代码框。
- 清理标题符号和分隔线。

## 主题切换

前台和后台右上角都有深色 / 亮色切换按钮。

主题会保存在浏览器中。

## v3.0 新功能使用教程

### 多 Agent 协作使用指南

v3.0 支持主智能体调度子智能体协同回答复杂问题，适用于需要多领域知识综合的场景。

操作步骤：

1. 在智能体后台编辑页，找到"子智能体"配置项，多选绑定需要协作的子智能体。
2. 保存智能体配置。
3. 在前台选择该主智能体，提出复杂问题。
4. 主 Agent 会自动分析问题，调度合适的子 Agent 执行子任务，并整合结果生成最终回答。

> 提示：递归调用深度限制为 3 层，避免无限嵌套。子智能体本身也可以绑定子智能体，形成层级协作。

### 工具调用链编排使用指南

v3.0 支持工具串行编排，多个工具按顺序执行，前一个工具的输出可作为下一个工具的输入。

操作步骤：

1. 在智能体后台编辑页，找到"工具链"配置区域。
2. 选择需要串联的工具，按执行顺序排序。
3. 为每个工具设置输入映射：
   - `$user_input`：使用用户原始输入。
   - `$prev_output`：使用上一个工具的输出。
4. 保存配置。
5. 在前台提问时，系统会自动按链式顺序调用工具，并将所有工具结果交给大模型生成回答。

> 提示：工具链在 LLM 生成前执行，工具结果会作为上下文参与模型推理。任意工具执行失败会中断链路。

### 智能体评测使用指南

v3.0 提供智能体评测功能，帮助量化评估智能体回答质量。

操作步骤：

1. 进入后台"智能体评测"页面（`/eval`）。
2. 创建评测数据集：
   - 填写输入问题。
   - 填写期望关键词（多个关键词用逗号分隔，用于判断回答是否命中要点）。
3. 选择待评测的智能体，触发评测。
4. 查看准确率报告，包括整体命中率和各用例结果。
5. 点击单条用例可查看详细结果，包括实际回答和关键词命中情况。

> 提示：评测数据集越完整，评估结果越准确。建议覆盖智能体的典型使用场景和边界情况。

### 文件上传对话使用指南

v3.0 支持在前台对话中上传文件，文件内容自动注入对话上下文。

操作步骤：

1. 在前台对话页，点击输入框旁边的 📎 按钮上传文件。
2. 支持的文件类型：
   - PDF（含扫描件，需启用 OCR）
   - TXT
   - MD
   - 图片（PNG/JPG）
3. 文件大小限制为 10MB。
4. 上传后文件内容会自动解析并注入对话上下文。
5. 发送消息后，文件会自动清空，下次对话需要重新上传。

> 提示：扫描件 PDF 需要 OCR 功能支持，请确认后端已启用 OCR（`OCR_ENABLED=true`）并安装 Tesseract 引擎。

### 工作流画布使用指南

v3.0 提供可视化工作流画布，支持通过节点编排构建复杂业务流程。

操作步骤：

1. 进入后台"工作流画布"页面（`/workflows`）。
2. 点击"创建工作流"，填写名称和描述。
3. 在画布中添加节点，支持 6 种节点类型：
   - `start`：流程起点，接收用户输入。
   - `agent`：智能体节点，调用指定智能体。
   - `tool`：工具节点，调用配置的工具。
   - `knowledge`：知识库节点，检索指定知识库。
   - `condition`：条件分支节点，根据表达式选择路径。
   - `end`：流程终点，输出结果。
4. 配置节点间的连线关系，条件分支节点需配置分支表达式。
5. 保存工作流。
6. 点击"执行"运行工作流，查看各节点的执行结果和输出。

> 提示：工作流执行采用拓扑排序，确保节点按依赖顺序执行。条件分支可根据上游节点输出动态选择路径。

### RAG 调优使用指南

v3.0 支持按知识库维度独立配置检索参数，精细化调优 RAG 检索效果。

操作步骤：

1. 进入后台"RAG 调优"页面（`/rag-tuning`）。
2. 选择需要调优的知识库。
3. 调整检索参数：
   - `chunk_size`：文档切块大小。
   - `overlap`：切块重叠字符数。
   - `top_k`：返回结果数量。
   - `score_threshold`：相似度阈值。
   - 向量权重 / 关键词权重：混合检索的权重配比。
4. 在"检索测试"区域输入测试问题，查看召回的文档片段。
5. 根据效果反复调整参数并测试，直到检索效果满意。
6. 点击"保存配置"，参数会绑定到该知识库，运行时自动生效。

> 提示：调优参数与知识库一一绑定，不同知识库可独立配置。建议使用典型问题作为测试用例，确保参数适配实际使用场景。

### WebSocket 模式使用指南

v3.0 在前台对话中新增 WebSocket 通信模式，支持双向实时通信。

操作步骤：

1. 在前台对话页，切换到"WebSocket 模式"（默认仍为 SSE 模式）。
2. 查看页面上的连接状态指示灯：
   - 绿色：已连接。
   - 黄色：连接中 / 重连中。
   - 红色：连接失败。
3. 连接成功后即可正常对话，体验与 SSE 模式一致（逐字流式输出）。
4. 网络异常断开时，客户端会自动重连，重连成功后恢复会话上下文。

> 提示：WebSocket 模式适合需要双向交互的场景（如文件上传进度反馈）。普通流式问答使用默认 SSE 模式即可。心跳保活机制会自动维持长连接，无需手动干预。

## 常见问题

### 模型测试 401

通常是 API Key 无效、过期或填错。

### 回答慢或复杂问题失败

后端和前端超时时间已经调长，但复杂模型仍可能需要等待。可以检查后端日志和模型服务状态。

### 前台没有智能体

先到后台创建并发布智能体。前台只展示已发布智能体。

### 知识库检索没有结果

如果问题过于宽泛，系统会避免返回低相关片段。可以换成更具体的问题，例如包含产品名、药品名、条款名或业务关键词。

### 对话上下文丢失

前台会话保存在浏览器 localStorage 和后端数据库中。如果清除浏览器数据或使用无痕模式，本地会话会丢失，但后端仍保留对话记录。

### 工具调用没有触发

检查工具的触发关键词是否与用户问题匹配。关键词匹配是运行时决定是否调用工具的主要依据。

### 智能体导入失败

确保导入的 JSON 文件是从 AgentPilot 导出的格式。手动编辑的 JSON 可能缺少必要字段。

### 反馈数据在哪里查看

管理员登录后台后，可在反馈统计页面查看各智能体的点赞/点踩数量。
