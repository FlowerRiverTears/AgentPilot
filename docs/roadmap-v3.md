# 第三版开发路线

## 目标

第三版将 AgentPilot 从单智能体平台升级为多智能体协作、可评测、支持文件交互的智能体工作台。

核心方向：

- 多 Agent 协作：主 Agent 调度子 Agent，分工处理复杂任务。
- 工具调用链编排：一个问题触发多个工具，串行编排，前序工具输出作为后序工具输入。
- 智能体评测：用测试集自动跑评测，输出准确率、召回率、耗时等指标。
- 文件上传对话：前台用户上传文件，智能体即时分析回答。

## 优先级

### P7 多 Agent 协作

- [x] Agent 配置增加子智能体绑定（sub_agent_ids）。
- [x] 运行时主 Agent 可调用子 Agent，子 Agent 结果整合到主回答。
- [x] 前端智能体编辑页增加子智能体多选 UI。
- [x] 运行步骤展示子 Agent 调用轨迹。

### P7 工具调用链编排

- [x] Agent 配置增加工具链编排（tool_chain），定义工具调用顺序。
- [x] 运行时按链式调用工具，前序工具输出注入后序工具输入。
- [x] 前端智能体编辑页增加工具链编排 UI。
- [x] 运行步骤展示工具链调用轨迹。

### P8 智能体评测

- [x] 评测数据集管理：创建数据集、上传测试用例（问题+期望回答）。
- [x] 评测执行接口：批量运行测试集，记录每条结果。
- [x] 评测指标计算：准确率（关键词匹配）、平均耗时、成功率。
- [x] 前端评测页面：数据集管理、触发评测、查看评测报告。

### P8 文件上传对话

- [x] 前台文件上传接口：支持 PDF、图片、TXT、Markdown。
- [x] 文件内容解析：复用 document_loader 提取文本。
- [x] 文件内容注入对话上下文：上传文件后，内容作为上下文传给智能体。
- [x] 前台对话页增加文件上传按钮和文件预览。

### P9 工作流画布

- [x] 工作流数据模型：Workflow + WorkflowRun 两张表。
- [x] 节点类型：start/agent/tool/knowledge/condition/end，按拓扑排序执行。
- [x] 工作流 CRUD + 执行接口（9 个 API）。
- [x] 前端工作流画布页面：节点编辑器、连线编辑器、执行测试。

### P9 RAG 调优面板

- [x] RagConfig 数据模型：按知识库维度存储检索参数。
- [x] 调优参数：chunk_size、chunk_overlap、top_k、score_threshold、向量/关键词权重。
- [x] 调优接口：获取/更新配置 + 检索测试（4 个 API）。
- [x] 前端 RAG 调优面板：滑块调参、实时检索测试。

### P9 WebSocket 实时通信

- [x] WebSocket 端点 `/ws/chat`：双向通信，SSE 事件转换。
- [x] 前端 PortalPage WebSocket 模式：自动重连、心跳保活、连接状态指示灯。
- [x] SSE/WebSocket 模式切换，默认 SSE 保持兼容。
- [x] 中途取消生成：WebSocket 模式下发送 `{"action": "cancel"}` 停止 LLM 流式输出。
- [x] 工具调用进度推送：`tool_progress` 事件，前端实时显示"正在调用工具..."。

### P9 向量数据库切换

- [x] 向量存储抽象层：VectorStoreBackend 抽象接口。
- [x] pgvector 后端：复用现有 PostgreSQL + pgvector。
- [x] Qdrant 后端：通过 REST API 调用 Qdrant。
- [x] 配置切换：VECTOR_DB_BACKEND 环境变量控制。

## 第三版验收流程

1. 管理员创建主智能体，绑定子智能体。
2. 前台提问复杂问题，主智能体调度子智能体协作回答。
3. 管理员配置工具链，一个问题串行调用多个工具。
4. 管理员上传评测数据集，触发评测，查看准确率报告。
5. 前台用户上传 PDF 文件，智能体基于文件内容回答问题。
6. 管理员创建工作流，编排多步骤节点，执行并查看结果。
7. 管理员在 RAG 调优面板调整检索参数，实时验证检索效果。
8. 前台切换 WebSocket 模式，体验双向实时通信。
9. 管理员配置 VECTOR_DB_BACKEND 切换向量数据库后端。

## 不进入第三版范围

- 多租户与计费。
- 插件市场。
