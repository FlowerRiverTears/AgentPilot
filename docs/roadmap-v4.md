# 第四版开发路线

## 目标

v2 完成了智能体平台的基础能力，v3 补齐了多 Agent 协作、工具链编排和评测系统。v4 的目标是将 AgentPilot 从"功能完善的开发工具"升级为"生产级企业智能体平台"。

核心方向：

- 安全体系落地：权限管控、审计追溯、API Key 管理
- 可观测性：日志、监控、追踪体系
- 质量保障：测试覆盖、错误处理规范化
- 性能优化：RAG 缓存、向量检索加速
- 架构升级：工作流画布
- 生态扩展：MCP 协议、插件市场、多租户

---

## 优先级

### P9 安全与权限体系落地 — 已完成

v2 设计了用户模型和角色字段，v3 在架构文档中规划了 RBAC，v4 现已完整落地。

- [x] **用户管理 CRUD**：管理员可创建/编辑/禁用/删除用户，分配角色（admin / user）
- [x] **RBAC 权限校验**：后端 `get_current_admin` 守卫 + 前端路由守卫 + 角色菜单过滤
- [x] **前端权限控制**：路由守卫根据角色隐藏/显示菜单项，非 admin 无法访问 `/users` 页面
- [x] **审计日志**：`audit_logs` 表 + 查询 API，记录用户管理操作（创建/编辑/删除），含操作人、IP、时间
- [x] **密码修改**：管理员可重置任意用户密码，用户可修改自己的密码（需验证旧密码）
- [ ] **API Key 管理**：前端提供 API Key 创建/查看/删除界面，支持设置过期时间和权限范围
- [ ] **密码策略增强**：支持密码强度校验、首次登录强制修改密码、密码过期提醒

**参考文档**：[用户权限体系设计](用户权限体系设计.md)

### P10 可观测性

当前系统使用 Python logging 输出到 stdout，trace_id 已生成但未充分利用。P10 建立完整的可观测性体系。

- [ ] **结构化日志**：统一日志格式（JSON），包含 trace_id、user_id、agent_id、耗时等字段
- [ ] **请求追踪**：trace_id 从前端请求贯穿到后端日志、LLM 调用、工具调用、数据库查询，全链路可追踪
- [ ] **运行监控面板**：后台新增 Dashboard 升级版，展示 QPS、P95 延迟、LLM Token 消耗、错误率、知识库检索命中率
- [ ] **告警机制**：LLM 调用失败率超阈值、Token 消耗异常增长时触发告警（企业微信/邮件/Webhook）
- [ ] **运行历史增强**：Run 详情页增加全链路时间线视图（知识库检索耗时、工具调用耗时、LLM 首 Token 延迟、总耗时）

### P11 测试与质量保障

当前后端和前端均无自动化测试。P11 建立测试体系，保障代码质量。

- [ ] **后端单元测试**：核心模块测试覆盖（auth、security、rag pipeline、tool registry、agent runtime）
- [ ] **后端集成测试**：API 接口测试（使用 pytest + httpx AsyncClient），覆盖所有路由
- [ ] **前端组件测试**：关键页面组件的单元测试（Vitest + Vue Test Utils）
- [ ] **E2E 测试**：核心用户流程的端到端测试（Playwright）
- [ ] **CI 流水线**：GitHub Actions / GitLab CI 自动运行 lint + test + typecheck
- [ ] **前端状态处理规范化**：统一加载态（n-spin）、空状态（n-empty）、错误态（n-result）组件，覆盖所有页面

### P12 性能优化

随知识库规模增长和并发量提升，P12 聚焦性能瓶颈。

- [ ] **RAG 检索缓存**：相同查询命中缓存直接返回，减少重复 Embedding 计算和向量检索
- [ ] **Embedding 批量处理**：文档上传时批量生成 Embedding（已实现 asyncio.gather），增加限流和重试机制
- [ ] **向量数据库切换**：支持 Milvus / Qdrant 作为 pgvector 的替代方案，大规模场景下性能更优
- [ ] **RAG 调优面板**：前端可视化调整 chunk_size、chunk_overlap、检索阈值、top_k 等参数，实时预览检索效果
- [ ] **数据库查询优化**：慢查询分析、索引优化、连接池调优
- [ ] **前端性能优化**：路由懒加载、组件按需引入、大列表虚拟滚动

### P13 实时通信与工作流

- [x] **WebSocket 实时通信**：~~替代当前 SSE 方案~~（已完成）— 双向通信、工具调用进度实时推送、用户可中途取消生成
- [ ] **工作流画布**：可视化编排多步骤 Agent 流程（拖拽节点、连线、条件分支），类 Dify/Coze 体验
- [ ] **工作流模板市场**：预置常见工作流模板（客服工单处理、数据分析报告、多源信息汇总），一键导入

### P14 生态与扩展

- [ ] **MCP 协议支持**：作为 MCP Client 接入外部 MCP Server 提供的工具，同时作为 MCP Server 对外暴露 Agent 能力
- [ ] **工具插件化架构**：重构 tools/registry.py，支持动态加载工具插件（Python 包、HTTP 服务、MCP Server），无需修改核心代码即可扩展
- [ ] **模型网关增强**：支持多模型路由（根据任务复杂度自动选择模型）、模型 fallback 链、Token 预算管理
- [ ] **Agentic RAG**：智能体自主决策检索策略（是否需要检索、检索哪些知识库、是否需要多轮检索），替代当前固定的检索流程

### P15 企业级特性

- [ ] **多租户隔离**：租户级数据隔离（数据库 Schema 或 tenant_id），租户管理员自管理用户和资源
- [ ] **用量统计与计费**：LLM Token 消耗、Embedding 调用次数、存储空间统计，支持额度限制和告警
- [ ] **插件市场**：社区贡献工具插件、Prompt 模板、工作流模板，支持评分和评论
- [ ] **国际化完善**：英文界面翻译覆盖、日期/数字本地化格式、RTL 语言支持
- [ ] **数据导出与迁移**：智能体配置、知识库、运行历史的一键导出和跨环境迁移

## v4 验收流程

1. 管理员创建 user 角色账号，user 登录后只能访问前台和运行历史，无法进入后台管理
2. 管理员在审计日志中查看所有用户操作记录
3. 监控面板实时展示 QPS、延迟、Token 消耗
4. 运行后端测试套件，覆盖率 > 70%
5. 上传 100+ 文档到知识库，检索延迟 < 500ms
6. 前台对话使用 WebSocket，打字机效果流畅，工具调用进度实时显示，支持中途取消（已完成）
7. 工作流画布拖拽编排"查订单 → 查库存 → 生成汇总报告"，一键执行
8. 接入外部 MCP Server 工具，Agent 调用第三方工具完成任务

## 不进入 v4 范围

- AI 自动生成 Agent 配置（根据自然语言描述自动创建 Agent）
- Agent 自优化闭环（根据评测结果自动调整 Prompt 和参数）
- 联邦学习 / 隐私计算
- 移动端 App
- 语音交互
