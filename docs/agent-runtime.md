# 智能体运行时

## 当前运行流程

入口：

```text
POST /api/runs
```

流程：

1. 根据 `agent_id` 读取智能体配置。
2. 根据用户输入检索智能体绑定的知识库，并过滤低质量和低相关片段。
3. 根据用户输入和工具绑定情况调用应用工具。
4. 拼接系统提示词、多轮历史、用户问题、知识库上下文、工具结果。
5. 调用模型网关生成回答。
6. 保存运行记录和执行步骤。
7. 返回回答、引用来源、执行过程和模型信息。

## 上下文结构

后端会把上下文组织成：

```text
用户问题：
...

知识库上下文：
...

应用工具结果：
...

请给出清晰回答。优先使用应用工具结果回答实时业务数据；
使用知识库内容时说明来源；如果没有可靠依据，请明确说明无法确认。
```

## 执行步骤

当前返回的步骤包括：

- 接收任务
- 知识库检索
- 应用工具调用
- 大模型生成

## 错误处理

当真实大模型调用失败时，后端会返回错误说明，提示用户检查：

- 模型接口地址
- API Token
- 模型名
- 后端服务

## 检索增强

- PDF 上传会先提取可读文本，再切块入库。
- 切块入库前会做可读性检测，丢弃二进制残留和乱码片段。
- 检索后会按最低相关度过滤结果，并对重复片段去重。
- 对中文短语加入关键词匹配兜底，降低本地 fallback embedding 对中文短语不稳定的影响。
- 如果没有可靠命中，知识库上下文为空，模型应明确说明无法确认，而不是基于弱相关内容回答。

## 前台上下文

`/portal` 会把最近会话和多轮上下文保存到浏览器 `localStorage`。刷新页面后会恢复消息列表和上下文，继续发送给后端用于多轮回答。

## 第二版增强

- Run 列表和 Run 详情页已完成。
- 引用来源和执行步骤已可在运行历史中回看。
- 流式输出已接入。
