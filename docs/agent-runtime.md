# 智能体运行时

## 当前运行流程

入口：

```text
POST /api/runs
```

流程：

1. 根据 `agent_id` 读取智能体配置。
2. 根据用户输入检索智能体绑定的知识库，并过滤低质量和低相关片段。
3. 根据用户输入和工具绑定情况调用应用工具。
4. 拼接系统提示词、多轮历史、用户问题、知识库上下文、工具结果。
5. 调用模型网关生成回答。
6. 保存运行记录和执行步骤。
7. 返回回答、引用来源、执行过程和模型信息。

## 上下文结构

后端会把上下文组织成：

```text
用户问题：
...

知识库上下文：
...

应用工具结果：
...

请给出清晰回答。优先使用应用工具结果回答实时业务数据；
使用知识库内容时说明来源；如果没有可靠依据，请明确说明无法确认。
```

## 执行步骤

当前返回的步骤包括：

- 接收任务
- 知识库检索
- 应用工具调用
- 大模型生成

## 错误处理

当真实大模型调用失败时，后端会返回错误说明，提示用户检查：

- 模型接口地址
- API Token
- 模型名
- 后端服务

流式运行失败时，run 记录的 status 设为 failed，error 字段记录异常信息。前端运行详情页展示失败原因和重新执行按钮。

## 检索增强

- PDF 上传会先提取可读文本，再切块入库。
- 切块入库前会做可读性检测，丢弃二进制残留和乱码片段。
- 检索后会按最低相关度过滤结果，并对重复片段去重。
- 对中文短语加入关键词匹配兜底，降低本地 fallback embedding 对中文短语不稳定的影响。
- 如果没有可靠命中，知识库上下文为空，模型应明确说明无法确认，而不是基于弱相关内容回答。

## 前台上下文

`/portal` 会把最近会话和多轮上下文保存到浏览器 `localStorage`。刷新页面后会恢复消息列表和上下文，继续发送给后端用于多轮回答。

## 对话记忆管理

前台 `/portal` 的对话记忆通过 `conversations` 表持久化：

- 每个会话由前端生成的 `session_id` 标识。
- 完整消息列表存储在 `messages` 字段（JSON）。
- 上下文压缩后的摘要存储在 `summary` 字段，`summary_to_turn` 记录摘要覆盖到第几轮。
- 支持按智能体过滤会话列表、搜索会话内容、删除会话。

## 反馈机制

用户可对智能体回答进行点赞/点踩：

- 反馈记录存储在 `feedbacks` 表，关联 `run_id` 和 `agent_id`。
- 支持匿名提交反馈（前台用户无需登录）。
- 后台可查看反馈列表和按智能体统计的点赞/点踩数量。

## 已实现的增强

- Run 列表和 Run 详情页已完成。
- 引用来源和执行步骤已可在运行历史中回看。
- 流式输出已接入。
- 工具调用参数由模型或规则生成。
- Run 详情页展示每一步耗时。
- 工具调用结果单独入库并在运行详情中展示。
- 失败重试：POST /runs/{run_id}/retry 接口，前端运行详情页展示"重新执行"按钮。
- 流式运行失败时正确记录 status=failed 和 error 信息。
- 工具调用日志：每次工具调用记录到 tool_calls 表，含耗时、状态码、输入输出。

## 已实现的增强（v0.2.0 新增）

- 对话记忆管理：查看历史会话列表、删除会话、会话搜索。接口：`GET/POST/DELETE /conversations`。
- 上下文压缩（滑动窗口+摘要）：长对话自动压缩早期上下文，Token 使用率达到上下文窗口 60% 时触发，保留最近 6 轮原文，早期对话由 LLM 生成摘要替换。摘要缓存在 `conversations` 表的 `summary` 和 `summary_to_turn` 字段中。
- 多知识库绑定：一个智能体可绑定多个知识库，检索时合并结果。
- 智能体导入/导出：`POST /agents/import` 和 `GET /agents/{agent_id}/export`。
- 回答反馈：用户对回答点赞/点踩，记录到 `feedbacks` 表，支持按智能体统计。
- Prompt 模板库：预置常用系统提示词模板，一键套用。

## 待实现的增强

- WebSocket 实时通信替代 SSE，支持双向通信。
- 工具调用链：一个问题触发多个工具，串行/并行编排。
